Hi judges, this is SevaGrid Command, a real-time volunteer operations command center built for the Smart Volunteer Deployment and Workforce Optimization track.

The core problem is that Mahakumbh operations are not just about counting volunteers. Thousands of people may be supporting crowd guidance, medical aid, lost-child response, language support, logistics, water relief, and security coordination at the same time. The hard question is: who should move, where should they go, and why is that the right decision right now?

SevaGrid answers that with a map-first command surface. The map is the primary workspace, not a traditional dashboard. Each sector node shows risk, coverage, and deployment status. Route lines show where volunteers are being moved. Incident beacons stay spatially attached to the sector where action is needed.

At the top, operators can change the live operating scenario, such as Snan Surge, Heat Alert, Missing Person, or Balanced mode. They can filter by skill, adjust crowd and heat pressure, and add a new incident. Every change recalculates the deployment plan.

On the right, the AI Dispatch panel explains recommendations in operational language. It ranks volunteers using skill fit, location, language coverage, mobility, fatigue, current workload, and incident match. This is important because optimization should not simply send the nearest person every time. It should protect tired volunteers and close the highest-risk gaps first.

The incident queue lets an operator add and send response tickets. When a live incident appears, the relevant sector risk and staffing demand update, and the recommendations adjust around that operational pressure.

The project is built with Next.js, React, TypeScript, Tailwind CSS, Framer Motion, and Vercel. The current demo uses typed local sample data so judges can inspect the model without needing credentials or API keys. The optimizer is deterministic and explainable in source code.

Security and validation are also documented. The app has no backend secrets, no real personal data, production security headers, HTTPS deployment, a passing production build, passing lint checks, and zero dependency audit vulnerabilities.

For the hackathon theme, SevaGrid covers the full workflow: modeling volunteer profiles, managing availability and fatigue, allocating people to sectors, monitoring map-connected incidents, and optimizing workforce deployment based on operational requirements.

The future roadmap is to connect real signup data, GIS route timing, supervisor acknowledgement, WhatsApp or SMS dispatch, role-based authentication, and real-time check-ins. But the current version is already a working product demo that shows how Mahakumbh volunteer coordination can become faster, safer, and more explainable.

Live deployment is available at sevagrid-command dot vercel dot app, and the full source code is on GitHub under AnuranjanJain slash Round2-Experthire.
